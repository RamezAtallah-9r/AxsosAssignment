8️⃣ Routing Inside Multiple Apps FOCUS

Django utilizes the explicit app_name configuration variable to wrap individual routing targets cleanly, protecting systems against route signature hijacking.

📄 App Path Strategy Mapping: app1/urls.py

from django.urls import path
from . import views

app_name = 'app1'

urlpatterns = [
    path('', views.index, name='index'),
    path('item/<int:item_id>/', views.detail, name='detail'),
    path('create/', views.create, name='create'),
]

🔗 Dynamic URL Reversing Matrix

Execution Context Syntax Implementation Structure
HTML Template Layouts {% url 'app1:detail' item_id=5 %}
Python View Inversion reverse('app1:detail', kwargs={'item_id': 5})
System Redirection Views return redirect('app2:dashboard')

⚠️ Core Architecture Absolute Law: Always call paths explicitly utilizing the namespaced layout paradigm (app_name:view_name). Absolute hardcoding of raw route expressions like /app1/item/5/ is strictly forbidden.

    if request.method == 'POST':
        title = request.POST.get('title')
        price = request.POST.get('price')

        if title and price:
            # Stash execution variable briefly into browser storage matrix
            request.session['last_submitted'] = title
            # Distribute feedback validation alert across redirection layers
            messages.success(request, f'Item reference "{title}" tracked successfully.')
            # Execute redirect to bypass double POST refresh submission mutations
            return redirect('app1:index')

    return render(request, 'app1/create.html')

1️⃣7️⃣ Altering Models After Creation

Understanding Model Migrations

After you create or modify a model, Django needs to track these changes through migrations. This ensures your database schema stays synchronized with your code.

📝 Step-by-Step: Adding a New Field to Existing Model

Original Model:

class Product(models.Model):
    name = models.CharField(max_length=100)
    price = models.DecimalField(max_digits=8, decimal_places=2)

Modified Model (Adding a field):

class Product(models.Model):
    name = models.CharField(max_length=100)
    price = models.DecimalField(max_digits=8, decimal_places=2)
    description = models.TextField(blank=True)  # New field added

Execute migrations:

python manage.py makemigrations
python manage.py migrate

Common Field Modification Operations

Operation Code Change Important Notes
Add New Field description = models.TextField(blank=True) Use blank=True if field already exists in database
Change Field Type Change CharField to TextField Django will handle type conversion on migration
Rename Field Edit field name, then python manage.py makemigrations --name rename_field For better control, manually edit migration file
Delete Field Remove the field line entirely Data in that field will be permanently deleted
Add Default Value status = models.CharField(max_length=20, default='active') Existing records will use the default value

Example: Altering a Model Comprehensively

class Product(models.Model):
    # Original fields
    name = models.CharField(max_length=100)
    price = models.DecimalField(max_digits=8, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)
    
    # New fields being added
    description = models.TextField(blank=True)
    stock = models.IntegerField(default=0)  # With default value
    status = models.CharField(max_length=20, choices=[
        ('active', 'Active'),
        ('inactive', 'Inactive'),
        ('discontinued', 'Discontinued')
    ], default='active')
    updated_at = models.DateTimeField(auto_now=True)
    
    def __str__(self):
        return self.name

1️⃣8️⃣ One-to-One Relationships

Understanding One-to-One Relationships

A one-to-one relationship exists when each record in one model is linked to exactly one record in another model. Example: Each user has exactly one profile.

Creating One-to-One Models

from django.db import models

class User(models.Model):
    username = models.CharField(max_length=100, unique=True)
    email = models.EmailField()
    
    def __str__(self):
        return self.username

class UserProfile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    bio = models.TextField(blank=True)
    age = models.IntegerField(null=True)
    avatar = models.ImageField(upload_to='avatars/', blank=True)
    
    def __str__(self):
        return f"Profile for {self.user.username}"

Explanation:

  • OneToOneField(User, on_delete=models.CASCADE) — Creates a one-to-one relationship
  • on_delete=models.CASCADE — If User is deleted, the Profile is deleted too
  • Use on_delete=models.PROTECT to prevent deletion if related record exists
  • Use on_delete=models.SET_NULL to set the field to NULL if related record is deleted

Creating & Accessing One-to-One Data

Creating a One-to-One Record:

# Method 1: Create user first, then profile
from app1.models import User, UserProfile

user = User.objects.create(username='john_doe', email='john@example.com')
profile = UserProfile.objects.create(user=user, bio='Software Developer', age=28)

# Method 2: Create both in one go
user = User.objects.create(username='jane_smith', email='jane@example.com')
UserProfile.objects.create(user=user, bio='Designer', age=25)

# Method 3: Using related object creation
user = User.objects.create(username='bob_jones', email='bob@example.com')
user.userprofile.bio = 'Project Manager'  # This only works if profile exists
user.userprofile.save()

Accessing One-to-One Related Data:

# Access profile from user
user = User.objects.get(username='john_doe')
print(user.userprofile.bio)    # Access the related profile
print(user.userprofile.age)    # Access profile field

# Access user from profile
profile = UserProfile.objects.get(user__username='john_doe')
print(profile.user.email)      # Access the related user
print(profile.user.username)

1️⃣9️⃣ One-to-Many Relationships

Understanding One-to-Many Relationships

A one-to-many relationship exists when one record in a model can be associated with multiple records in another model. Example: One author writes many books.

Creating One-to-Many Models

from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField()
    
    def __str__(self):
        return self.name

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(Author, on_delete=models.CASCADE, related_name='books')
    publication_date = models.DateField()
    price = models.DecimalField(max_digits=6, decimal_places=2)
    
    def __str__(self):
        return self.title

Key Components:

  • ForeignKey(Author, ...) — Establishes the one-to-many relationship from Book to Author
  • on_delete=models.CASCADE — If Author is deleted, all their Books are deleted
  • related_name='books' — Allows reverse lookup (explained in next section)

Creating One-to-Many Records

from app1.models import Author, Book
from datetime import date

# Create an author
author = Author.objects.create(name='J.K. Rowling', email='jk@example.com')

# Create multiple books for the author
Book.objects.create(
    title='Harry Potter and the Philosopher\'s Stone',
    author=author,
    publication_date=date(1997, 6, 26),
    price=15.99
)
Book.objects.create(
    title='Harry Potter and the Chamber of Secrets',
    author=author,
    publication_date=date(1998, 7, 2),
    price=16.99
)

# Alternative: Using the author variable directly
book = Book(
    title='Harry Potter and the Prisoner of Azkaban',
    author=author,
    publication_date=date(1999, 7, 8),
    price=17.99
)
book.save()

Accessing One-to-Many Data

Forward Relationship (Book to Author):

# Get a book and access its author
book = Book.objects.get(title='Harry Potter and the Philosopher\'s Stone')
print(book.author.name)        # Output: J.K. Rowling
print(book.author.email)

2️⃣0️⃣ Reverse Lookup with related_name

Understanding related_name

related_name allows you to query the relationship from the "one" side (Author) to access the "many" side (Books). Without it, you'd use the lowercased model name with _set.

Reverse Lookups Using related_name

# With related_name='books' defined in the ForeignKey
author = Author.objects.get(name='J.K. Rowling')

# Access all books by this author
all_books = author.books.all()              # Returns all books
print(all_books.count())                    # Count of books

# Iterate through books
for book in author.books.all():
    print(f"{book.title} - ${book.price}")

# Get first and last book
first_book = author.books.first()
last_book = author.books.last()

# Filter books with additional conditions
recent_books = author.books.filter(publication_date__year__gte=2000)
expensive_books = author.books.filter(price__gt=20)

# Check if author has any books
has_books = author.books.exists()
num_books = author.books.count()

Without related_name (Django Default):

# If you didn't define related_name, Django creates a default one
# Default format: lowercase_modelname_set
author = Author.objects.get(name='J.K. Rowling')
books = author.book_set.all()  # Less readable than using related_name

2️⃣1️⃣ Many-to-Many Relationships

Understanding Many-to-Many Relationships

A many-to-many relationship exists when records in one model can be associated with multiple records in another model, and vice versa. Example: A student takes many courses, and a course has many students.

Creating Many-to-Many Models

from django.db import models

class Student(models.Model):
    name = models.CharField(max_length=100)
    student_id = models.CharField(max_length=20, unique=True)
    
    def __str__(self):
        return self.name

class Course(models.Model):
    title = models.CharField(max_length=200)
    code = models.CharField(max_length=10, unique=True)
    instructor = models.CharField(max_length=100)
    students = models.ManyToManyField(Student, related_name='courses')
    
    def __str__(self):
        return self.title

Key Points:

  • ManyToManyField(Student, ...) — Creates a many-to-many relationship
  • Django automatically creates a junction table behind the scenes
  • related_name='courses' — Allows reverse queries from Student

Creating Many-to-Many Relationships

Method 1: Using add()

from app1.models import Student, Course

# Create objects
student1 = Student.objects.create(name='Alice Smith', student_id='S001')
student2 = Student.objects.create(name='Bob Johnson', student_id='S002')
course = Course.objects.create(title='Python Programming', code='CS101', instructor='Dr. Brown')

# Add students to course (associate them)
course.students.add(student1)
course.students.add(student2)

# Or add multiple at once
course.students.add(student1, student2)

Method 2: Using set()

# Replace all relationships
course.students.set([student1, student2])  # Only these two students in this course

Method 3: Using create()

# Create new object and add in one step
course.students.create(name='Charlie Davis', student_id='S003')

Accessing Many-to-Many Data

From Course to Students (Forward):

course = Course.objects.get(code='CS101')

# Get all students in course
all_students = course.students.all()

# Count students
num_students = course.students.count()

# Filter students
math_students = course.students.filter(name__contains='math')

# Check if student is enrolled
is_enrolled = course.students.filter(name='Alice Smith').exists()

# Get first student
first_student = course.students.first()

From Student to Courses (Reverse):

student = Student.objects.get(name='Alice Smith')

# Get all courses for student
all_courses = student.courses.all()

# Count courses
num_courses = student.courses.count()

# Filter courses
python_courses = student.courses.filter(title__contains='Python')

# Check if enrolled in specific course
is_enrolled = student.courses.filter(code='CS101').exists()

Removing Many-to-Many Relationships

course = Course.objects.get(code='CS101')
student = Student.objects.get(name='Alice Smith')

# Remove single student
course.students.remove(student)

# Remove multiple students
course.students.remove(student1, student2)

# Remove all students
course.students.clear()

# Alternative: remove using filter
course.students.set(course.students.exclude(name='Alice Smith'))

2️⃣3️⃣ Filtering, Ordering & Fetching Data

Filtering Data

Basic filter():

from app1.models import Product

# Exact match
cheap_products = Product.objects.filter(price__lt=50)

# Multiple conditions (AND logic)
results = Product.objects.filter(price__gt=100, stock__gt=0)

# OR condition (Q objects)
from django.db.models import Q
results = Product.objects.filter(Q(price__lt=50) | Q(stock=0))

Field Lookup Operators:

Operator Syntax Example Meaning
Exact (default) field=value filter(name='Laptop') Exact match
Iexact field__iexact filter(name__iexact='LAPTOP') Case-insensitive exact match
Contains field__contains filter(name__contains='top') Contains substring
Icontains field__icontains filter(name__icontains='TOP') Case-insensitive contains
Greater than field__gt filter(price__gt=100) Price > 100
Greater or equal field__gte filter(price__gte=100) Price >= 100
Less than field__lt filter(price__lt=100) Price < 100
Less or equal field__lte filter(price__lte=100) Price <= 100
In list field__in filter(status__in=['active','pending']) Status is in list
Range field__range filter(price__range=[50,200]) Price between 50-200
Startswith field__startswith filter(name__startswith='Pro') Name starts with 'Pro'
Endswith field__endswith filter(name__endswith='Pro') Name ends with 'Pro'

Ordering Data

Order by single field:

# Ascending order (A to Z, 0 to 9)
products = Product.objects.all().order_by('name')

# Descending order (Z to A, 9 to 0) - use minus sign
products = Product.objects.all().order_by('-price')

# Order by date (oldest first)
products = Product.objects.all().order_by('created_at')

# Order by date (newest first)
products = Product.objects.all().order_by('-created_at')

Order by multiple fields:

# Primary: by status, Secondary: by price
products = Product.objects.order_by('status', '-price')

# Highest price first, then by name alphabetically
products = Product.objects.order_by('-price', 'name')

Chaining Filters and Ordering

# Filter AND Order combined
expensive_in_stock = Product.objects.filter(
    price__gt=500,
    stock__gt=0
).order_by('-price')

# Multiple filters (AND logic)
results = Product.objects.filter(
    price__gte=100
).filter(
    stock__gt=0
).order_by('name')

# With exclude (NOT logic)
not_discontinued = Product.objects.exclude(
    status='discontinued'
).order_by('-created_at')

Advanced Filtering with Q Objects

OR conditions:

from django.db.models import Q

# Products that are either cheap OR out of stock
results = Product.objects.filter(
    Q(price__lt=50) | Q(stock=0)
)

# Complex: (cheap AND in stock) OR (expensive)
results = Product.objects.filter(
    Q(price__lt=50, stock__gt=0) | Q(price__gt=500)
)

NOT conditions:

from django.db.models import Q

# Products that are NOT discontinued
active = Product.objects.filter(~Q(status='discontinued'))

# NOT (cheap AND in stock)
results = Product.objects.filter(
    ~Q(price__lt=50, stock__gt=0)
)

from django.shortcuts import render, get_object_or_404
from django.db.models import Q
from app1.models import Product, Author, Book

def product_list(request):
    # Get all products ordered by newest first
    products = Product.objects.all().order_by('-created_at')
    
    # Pass to template
    return render(request, 'app1/product_list.html', {
        'products': products,
        'total_count': products.count()
    })

def filtered_products(request):
    # Get filter parameters from URL or form
    min_price = request.GET.get('min_price', 0)
    max_price = request.GET.get('max_price', 10000)
    search = request.GET.get('search', '')
    
    # Build complex filter
    products = Product.objects.filter(
        Q(price__gte=min_price) &
        Q(price__lte=max_price) &
        Q(name__icontains=search)
    ).order_by('-created_at')
    
    return render(request, 'app1/filtered_list.html', {
        'products': products,
        'search_term': search
    })

def author_books(request, author_id):
    # Get author with related books
    author = get_object_or_404(Author, pk=author_id)
    books = author.books.all().order_by('-publication_date')
    
    return render(request, 'app1/author_detail.html', {
        'author': author,
        'books': books,
        'book_count': books.count()
    })

from django.core.paginator import Paginator
from django.shortcuts import render
from app1.models import Product

def product_list_paginated(request):
